## 검증 명제

> "모든 간접투자의 엔드스테이션은 환율이다"

## 검증 결과: ✅ 적합

### 1. 논리 구조

```
개별종목 분석 (N개): 기업 × 산업 × 실적 × 거시변수 = 관리비용 큼
환율 분석 (1개):   Fed정책 + 헤게모니수수료 = 관리비용 작음

정보비율 = 얻는 정보 / 들이는 노력 → 환율 쪽이 압도적
```

### 2. 백서와의 정합성

| 층위 | 백서 연결점 |
|------|------------|
| 공통분모 명제 | Ch12 Rolling Forecast — 모든 통화노출은 결국 환율로 귀결 |
| N→2 환원 | Ch7.4~7.6 Dual-Unit Reduction Model (Module 7) |
| 연산효율 | Ch8 — 10개국 컷오프 = materiality threshold |
| 의사결정 | Ch13 — 베타헤지 전략, 개별자산 분석 대신 환율 기반 DCA |
| 스코프 | Ch7.3 — 학술모델 vs 실행모델 구분: 실행모델은 N=1로 충분 |

### 3. 의사결정 일관성

```
시장을 이기겠다는 욕심(알파)    → ❌ 이 전략의 목적 아님
구조적 손실만 방어하겠다(베타헤지) → ✅ Ch13.6 "알파가 아니라 베타 헤지"
가장 적은 관리비용으로 운용       → ✅ Ch6/8 연산효율 원칙
```

### 4. 가드레일 — "0이 되면 안 되는 지점" (2026.06.30 박씨 지적 반영)

```
End-Station = FX 명제는 "로데이터를 완전히 무시한다"는 의미가 아니다.
정확한 의미는:
  산업/경상수지/섹터 데이터 = 직접 베팅 대상 (X)
  산업/경상수지/섹터 데이터 = 환율 신호 신뢰도 검증 입력 (O)
```

**위험 시나리오**: 이 명제를 "경제학 모든 드라이버가 필요없다"로 확장하면 GATE 0(14장)이 통째로 무력화된다. GATE 0의 존재 이유 자체가 "로데이터 안에 숨은 의도(경상수지 발표 타이밍, 산업통계 조작 여부)를 역검증한다"는 것이기 때문.

**올바른 계층 정리**:
```
[기존 오해]                          [정정]
개별 드라이버 = 버려도 되는 노이즈    개별 드라이버 = 환율 모델에 먹이는 사료
환율 = 유일한 진짜 변수               환율 = 최종 정산층 (사료의 품질은 GATE 0이 검증)
```

**결론**: "안 본다"가 아니라 **"보긴 보는데, 그 자체로 투자판단을 안 하고 환율 모델의 입력 검증용으로만 쓴다"**가 정확한 위치. 완전히 0으로 만들면 7.4절 Dual-Unit 모듈(GATE 0이 경상수지 발표를 인센티브분석하는 부분)이 통째로 의미를 잃는다.

### 5. 최종 판정 (가드레일 포함)

> End-Station = FX 명제는 **최종 의사결정 레이어**에서 100% 맞다.
> 그러나 아래 산업/국가단위 데이터는 "필요없다"가 아니라 **"그 자체로 베팅하지 않고, 환율 모델에 먹이는 사료로만 쓴다"**는 계층 재정렬로 보정해야 완전히 일관된다.
> 완전히 버리는 게 아니라 계층을 낮춘 것 — 이 가드레일은 FX Engine의 모든 모듈에 적용된다.
