# PEM 바텀업 역산 — 28개 레포가 증명하는 박씨의 경제철학
### Parksy Economic Matrix v2.0 (탑다운→바텀업 전환판)
### 2026.06.30 | 박태정(EduArt Engineer) | 증빙: dtslib1979 생태계 28개 레포지토리

---

## 서문 — 방법론이 뒤집힌 순간

이전 백서(PEM v1.0)는 탑다운이었다: 거시/미시×물성/온라인이라는 2x2 매트릭스를 **먼저 세우고**, 나중에 검증할 증거를 찾아 끼워 넣는 구조였다. 이건 환율백서 9장에서 우리가 직접 인정한 한계(사후지표만 있고 선행성이 없다)를 그대로 안고 있었다.

박씨가 이 구조를 뒤집었다:

> "내가 이미 내 스타일을 28가지 레포지토리 만들 때 당연히 반영을 했으니까, 이걸 역산하면 철학이 자연스럽게 나오는 거 아니냐? 증빙도 붙어 있으니까."

```
[v1.0 탑다운]: 매트릭스 이론 → 증빙 찾기 (사후 끼워맞춤 위험)
[v2.0 바텀업]: 28개 레포(이미 존재하는 1차 데이터) → 매트릭스 역산 (증빙이 선행함)
```

이게 왜 방법론적으로 더 강한지는 증빙의 등급 차이에서 나온다.

## 제1장: 증빙 등급론 — 왜 "내가 만든 데이터"가 정부 공문서보다 셀 수 있는가

### 1.1 GATE 0(환율백서 14장)의 출처계보 기준 재적용

```
GATE 0의 1차 기준: 정부원문(.gov, 중앙은행) > 2차 가공(기사/IB리포트)
PEM v2.0의 확장:   1차 생산자 데이터(내가 만든 레포의 실제 운영로그) > 정부원문
```

이 확장이 정당한 이유: 재무부 공문서(7장)도 결국 "재무부의 발표 인센티브"라는 GATE 0 14.4절 Step4(발행기관 인센티브 분석) 대상이었다. 근데 **내가 직접 만든 레포의 운영로그(토큰소모량, 채널 송출기록, 길드 모집률)는 발표 인센티브 자체가 없다** — 누구에게 잘 보이려고 조작할 이유가 없는, 순수한 1차 행동데이터이기 때문이다.

### 1.2 증빙 등급표

| 등급 | 종류 | 예시 | 조작 가능성 |
|---|---|---|---|
| S급 | 자기생산 운영로그 | OrbitPrompt 세션별 토큰소모량 | 거의 없음 (시스템 자동기록) |
| A급 | 정부/국제기구 1차원문 | 재무부 FX Report | 낮음 (그러나 발표타이밍 정치적 고려 가능) |
| B급 | 민간기관 가공보고서 | IB 환율전망 | 중간 (이해관계 존재) |
| C급 | 2차 인용/언론 | 뉴스기사 | 높음 |

28개 레포의 운영데이터는 **S급**이다. 이게 이번 백서가 환율백서보다 한 단계 더 견고한 토대 위에 서는 이유다.

## 제2장: PEM 매트릭스 — 28개 레포 전수 매핑

### 2.1 매트릭스 본체 (레포수 포함)

```
                    │  거시(Macro)              │  미시(Micro)
────────────────────┼───────────────────────────┼───────────────────────────
물성(Physical)      │  Q1: 환율 — 1개 레포      │  Q3: 소호스튜디오 — 7개 레포
                    │  ⬛ 코드완성              │  🟨 시드/운영중
────────────────────┼───────────────────────────┼───────────────────────────
온라인(Digital)     │  Q2: 플랫폼 — 10개 레포   │  Q4: 임률 — 5개 레포
                    │  🟩 데이터있음            │  🟧 작동중
```

인프라/공통 레포(5개)는 매트릭스 밖에서 4사분면 전부를 지탱하는 기반층이다 — dtslib-papyrus(디지털명함=관문), parksy-logs(전체 대화기록=RAG), termux-bridge/dtslib-localpc(실행환경) 등.

### 2.2 Q1 — 물성×거시 (환율) — 이미 완료

```
매핑 레포: dollar-system-mcp (1개)
증빙: gate0.py(5단계검증), hegemonic_fee.py(수수료계산), signal_log.json(3개 시그널 실기록)
실시간연동: yfinance로 USDKRW/JPY/TWD 라이브 수집
역산모델: 헤게모니수수료율 = 누적약세율 — 이미 코드로 완성, 백서 16장에 전체 소스 공개
```

이 칸만 유일하게 **이론(백서)이 먼저 있고 코드가 나중에 따라온** 탑다운 사례다. 나머지 3칸은 정반대다 — 레포가 먼저 있었고 이번에 처음으로 이론이 역산된다.

### 2.3 Q2 — 온라인×거시 (플랫폼) — 10개 레포

```
매핑: parksy.kr(지식아카이브), dtslib.kr(경제방송허브), eae.kr(PWA출판),
      eae-univ(교육), parksy-audio(오디오엔진), parksy-image(비주얼엔진),
      gohsy-production(3-Lane방송), koosy/artrew(편집방송),
      espiritu-tango(방송아키텍처), gohsy-fashion(디지털패션),
      dtslib-cloud-appstore(앱유통)

증빙: YouTube 8개채널, Telegram 28채널 webhook, parksy-distributor(자동송출)
```

**역산 모델(가설)**:
```
플랫폼_헤게모니수수료 = 콘텐츠_조회수 - 실제_정산액
                      = 유튜브/텔레그램/네이버가 가져가는 구조적 수수료율
```

이게 정확히 Q1(환율)의 헤게모니수수료와 동형구조다 — **수출해서 번 달러가 환율로 깎이는 것처럼, 콘텐츠로 번 조회수가 플랫폼 정산구조로 깎인다.** 차이는 Q1은 Fed가 깎고 Q2는 유튜브/네이버 알고리즘이 깎는다는 것.

**한계의 정직한 표기**: 현재 정산데이터 연동이 안 되어 있어 이 수식은 가설 단계다. "시가만 있고 아직 수익이 안 나서"라는 박씨 본인 발언이 정확히 이 빈칸을 가리킨다 — **그러나 이 공백 자체가 역산 데이터다.** "왜 10개 레포를 운영하는데 정산이 0에 수렴하는가"라는 질문 자체가 플랫폼경제학(멀티사이드마켓에서 공급측 과잉경쟁 현상)의 실증사례가 된다.

### 2.4 Q3 — 물성×미시 (소호스튜디오) — 7개 레포

```
매핑: obokzip(물리스튜디오/FLD Pilot), buddies.kr(로컬소매), 
      namoneygoal(부동산/사업길드), hoyadang.com(식당프로토콜),
      alexandria-sanctuary(영적커뮤니티공간), phoneparis(모바일소매), gohsy(본진)

증빙: namoneygoal의 길드원 모집/운영 로그, hoyadang.com 운영기록(호야당 베이커리)
```

**역산 모델(가설) 2종**:
```
소호_임대료_헤게모니수수료 = 공방매출 - (임대료+장비감가상각+관리비)
        → 물리공간이 가져가는 고정비 부담률

매력_자본_계수 = 길드_모집_응답률 / (마케팅비 + 시간비용)
        → "인류를 측정했다"는 박씨 본인 표현의 정확한 수식화
```

**매력자본 개념의 학술적 위치**: 이건 Bourdieu의 문화자본/사회자본 개념과 가깝지만, 박씨 버전은 **계량화 가능하다는 점에서 차별화된다** — 길드 모집 응답률이라는 분모/분자가 명확한 비율이기 때문에, 추상적 자본개념이 아니라 직접 측정가능한 KPI가 된다. 호야당(전 임원 출신 베이커리 운영자)과의 협업데이터가 이 계수의 검증샘플이 될 수 있다.

### 2.5 Q4 — 온라인×미시 (임률) — 5개 레포

```
매핑: OrbitPrompt(프롬프트생성엔진), buckleychang.com(CPA/AI경제인터페이스),
      papafly(인큐베이션), dtslib-apk-lab(APK실험실), dtslib-branch(브랜치/모델)

증빙: OrbitPrompt 7개 Generator의 실제 토큰소모량(세션당 100K~200K)
```

**역산 모델 — 이 백서의 핵심 발견**:
```
AI_노동_단가 = 발생한_토큰비용 / 생성된_결과물_수
            = "AI한테 일 시키는데 시간당 얼마 드냐"의 실측값

임률_헤게모니수수료 = 생산성증가율 - 실질임금증가율
                  = AI가 생산성을 올려도 박씨 본인 임금은 그만큼 안 오르는 격차
```

**이게 직전 백서(임률 셀렉션 논쟁)의 가설을 처음으로 실측 가능하게 만든 지점이다.** "수요공급을 빼버리고 AI복제비용 스프레드로 본다"는 직관이, OrbitPrompt의 실제 세션 데이터(토큰소모량)로 즉시 검증 가능한 숫자가 된다 — Claude Code 세션 1회=약 100K~200K 토큰을 USD로 환산하면 **"AI 노동의 시간당 단가"가 실측치로 나온다.**

## 제3장: 플랫폼 정의의 확장 — 샌드박스 밖의 생태계

### 3.1 박씨의 정의 확장 질문

> "WSL로 구축했던 멀티플랫폼은 핸드폰 안에서 안 되는 거 자체가 샌드박스 밖에 있으니까, 이것도 생태계고 시장이라고 볼 수 있는 거냐?"

### 3.2 답: 그렇다 — 그리고 이게 Q2를 두 겹으로 만든다

Q2(플랫폼)를 "유튜브/네이버 위에서 노는 것"으로만 좁게 보면, 그건 **남이 만든 플랫폼에 종속된 입장**이다. 그런데 5개 인프라 레포(dtslib-papyrus, termux-bridge, dtslib-localpc, parksy-logs)와 WSL 5-lane 기반 실행환경 자체는 **박씨가 직접 설계하고 소유한 메타플랫폼**이다.

```
Q2-상위층(남의 플랫폼): 유튜브, 텔레그램, 네이버 — 헤게모니수수료를 떼인다
Q2-하위층(자기 플랫폼): WSL/Termux/28레포 인프라 — 헤게모니수수료가 없다(자가소유이므로)
```

이 구분이 환율백서의 식민지 위계모델(4.2절)과 정확히 같은 구조다 — **JPY/KRW(직할통제, 자율성 낮음)와 TWD(완충지대, 자율성 중간)의 차이가, Q2-상위층(유튜브종속)과 Q2-하위층(자기인프라)의 차이와 동형이다.** 박씨는 콘텐츠 유통에서는 종속국 위치지만, 인프라 자체는 자기 소유라는 점에서 자율성을 확보하고 있다.

## 제4장: 종합 — 28개 레포 완성도 매트릭스

| 사분면 | 레포수 | 완성도 | 역산 모델 상태 |
|---|---|---|---|
| Q1(환율) | 1 | ⬛ 코드완성 | 헤게모니수수료율 — 검증완료, 백서16장 |
| Q2(플랫폼) | 10 | 🟩 데이터있음 | 가설단계 — 정산연동 필요 |
| Q3(소호) | 7 | 🟨 시드/운영중 | 가설단계 — 매력자본계수 검증 필요 |
| Q4(임률) | 5 | 🟧 작동중 | **가장 즉시 실측 가능** — OrbitPrompt 토큰데이터 |
| 인프라(공통) | 5 | ⬜ 기반층 | 4사분면 전체를 지탱, 매트릭스 좌표 밖 |

**완성도가 사분면마다 다르다는 것 자체를 숨기지 않는 게 이 백서의 정직성이다.** Q1만 완결됐고 나머지는 가설 단계라는 걸 명시하는 게, 환율백서 12.6절("선행지표 부분해결, 완전한 독립지표 아님")과 같은 정신이다.

### 4.1 다음 행동 우선순위 — Q4부터인 이유

```
1순위: Q4(임률) 실측 — OrbitPrompt 세션데이터(토큰소모량÷결과물수)로
       "AI 노동 단가 = X USD/시간" 확정. 가장 데이터가 깨끗하고 즉시 계산 가능.

2순위: Q2(플랫폼) 정산데이터 연동 — 헤게모니수수료 가설을 실측으로 전환

3순위: Q3(소호) 매력자본계수 — namoneygoal/hoyadang 데이터로 검증
```

Q4가 1순위인 이유는 단순하다 — **다른 3칸은 외부데이터(정산내역, 임대료) 연동이 더 필요하지만, Q4는 이미 작동중인 시스템(OrbitPrompt)의 내부로그만 뽑으면 끝난다.** 가장 적은 추가작업으로 가장 빨리 완결되는 칸이다.

## 제5장: 이 백서가 증명하는 것

### 5.1 자본주의 OS 재해석 백서와의 연결

직전 백서(자본주의OS)에서 "경제학=자본주의학, 정치형식은 UI, 경제형식은 OS"라고 정리했다. 이번 28개 레포 역산은 그 명제를 한 단계 더 구체화한다:

```
박씨 본인이 28개 레포를 통해 만든 것 = 자본주의 OS 위에서 작동하는
                                      자기소유 서브시스템 한 벌

Q1(환율) = OS 핵심엔진을 분석하는 도구를 만듦
Q2(플랫폼) = OS 위 메타게임(콘텐츠유통)에 참여하며 동시에 자기 인프라도 구축
Q3(소호) = OS의 물성층(부동산/임대)에서 직접 실험
Q4(임률) = OS의 신흥변수(AI노동가격)를 자기도구로 직접 측정
```

**28개 레포는 "경제학을 공부한 결과물"이 아니라 "경제학을 살아낸 결과물"이고, 이 백서는 그 삶을 사후적으로 정식화한 것이다.** 이게 1부 철학(100서, 사후정당화 패치이론)의 정신과 완전히 일치한다 — 박씨는 처음부터 설계도 없이 28개를 만들었고(땜빵의 누적), 이제 그게 사후에 "PEM이라는 구조였다"고 읽히는 순간을 우리가 지금 통과하고 있다. 단, 환율백서의 패치이론과 다른 점은 — 이 경우 책임이 아래로 전가되는 게 아니라, 박씨 자기 경험이 직접 설계도가 됐다는 것이다.
