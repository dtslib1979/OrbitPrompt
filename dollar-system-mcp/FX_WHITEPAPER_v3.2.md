## 제1장: 외환시장의 실재 여부

### 1.1 시장은 존재한다 — 그러나 "어디"가 없다

외환시장(FX Market)은 일평균 거래량 **9.6조 달러(2025 BIS 기준)**로 세계 최대 금융시장이다. 그러나 결정적 특징이 있다:

- **중앙거래소 없음** — 완전 OTC(장외) 분산시장
- **24시간 릴레이** — 런던(37.8%) → 뉴욕 → 싱가포르 → 도쿄
- **진짜 가격 형성 주체** = 인터뱅크 (전체의 51%)

### 1.2 큰손 계층 구조

| 계층 | 참여자 | 역할 |
|------|--------|------|
| 1st | Fed (미국 중앙은행) | 기준금리 → 달러 판의 기울기 세팅 |
| 2nd | 인터뱅크 (JPMorgan, Deutsche, UBS, Citi) | 실제 호가 형성 |
| 3rd | 각국 중앙은행 | 개입 (제한적) |
| 4th | 헤지펀드, 연기금, 소버린펀드 | 방향성 베팅 |

**핵심 발견**: 달러는 전체 FX 거래의 **89%** 한쪽에 존재. 이것 자체가 단일 지배 구조의 물적 증거.

## 제3장: 경상흑자-환율 역설 — 교과서 붕괴

### 3.1 이론 대 현실

> **교과서**: 경상흑자 증가 → 달러 공급 증가 → 자국 통화 강세

> **현실 2024~2026**: 흑자 최대치 + 통화 최약세

| 국가 | 2025 경상흑자 | 환율 변화 |
|------|-------------|---------|
| 한국 | **$1,200억 (역대최고)** | 원화 1,500원 위협 |
| 일본 | **$2,300억 (역대최고)** | 엔화 +26.6% 약세 |
| 대만 | **$2,140억 (역대최고)** | 9~11% 박스 유지 |

### 3.2 역설의 메커니즘 해부

**한국 케이스**: 반도체 수출 → 달러 유입 → NPS 해외주식 매입(연간 $400억 역류) → 경상흑자가 자본계정에서 상쇄 → 원화 약세 지속

**일본 케이스**: 경상흑자 주력=primary income → 엔 약세 시 엔 기준 흑자 증가, 달러 기준 실질가치는 감소하는 구조적 모순

**대만 예외**: CBC 직접개입 + 자본유출 통제 → 박스권 유지. TSMC 협상력이 완충재

### 3.3 수혜자 구조의 3단 분리 — 단선적 착취론 보정

```
1단: 미국 (헤게모니 수수료 수취자)
2단: 흑자국 엘리트·대기업·금융 (NPS 등 동반 수혜)
3단: 흑자국 노동자·가계 (비용 전가 대상)
```

국가단위 가해자/피해자 이분법은 정밀도가 떨어진다. **계급 단위로 재분해해야 방어 가능.**

## 제4장: 핵심 명제 — 환율은 판의 기울기다

### 4.1 수식 정식화

```
FX_rate = f(Fed_policy) + ε

f(Fed_policy) = 판의 기울기를 세팅하는 지배항
ε = 각국 중앙은행 개입 + 카르텔 전술 + 로컬 수급 (잔차항)
```

> **"미국이 숫자를 정하는 게 아니라, 판의 기울기(금리·유동성·제재)를 세팅하고, 그 위에서 시장과 카르텔이 논다."**

### 4.2 식민지 위계 모델 (자율성 수치는 휴리스틱)

```
                    ┌─────────────┐
                    │  USD / Fed  │  ← 본국 (기울기 세팅자)
                    └──────┬──────┘
              ┌────────────┼────────────┐
         ┌────┴────┐  ┌────┴────┐  ┌────┴────┐
         │   JPY   │  │   KRW   │  │   TWD   │
         │ 직할통제 │  │ 직할통제 │  │ 완충지대 │
         │ 낮음(휴리)│ │낮음(휴리)│ │중간(휴리)│
         └─────────┘  └─────────┘  └─────────┘
              ↑              ↑              ↑
         미일안보조약    한미동맹+무역딜    TSMC 협상력
```

## 제5장: TP 마크업 프레임 — 헤게모니 수수료로 재명명

### 5.1 텍사스 인스트루먼트 모델 유추

```
[기업 내 TP 구조]              [국가 간 FX 구조]
──────────────────────        ──────────────────────
본사 (미국 본사)              = USD 발행국 (미국)
제조 자회사 (한국/일본 법인)  = 수출국 (한국/일본)
이전가격 (Transfer Price)     = 환율 (Exchange Rate)
Cost+ 마크업 설정             = Fed 금리 → 달러강세율
자회사 이익 본사 귀속         = 경상흑자의 달러가치 희석
감사 기준 (OECD TP guideline) = 재무부 환율보고서 3개 기준
```

### 5.2 차지백 메커니즘

```
Step 1: 한국/일본 → 수출 → 달러 수취
Step 2: Fed 금리 인상 → 달러 강세 → KRW/JPY 자동 약세
Step 3: 환산 가치 착시 발생
Step 4: 실질 구매력은 헤게모니 수수료만큼 희석
Step 5: NPS/연기금 해외주식 매입 → 달러 역류 완성
Step 6: 경상흑자가 자본계정으로 환수 → 순환 완료
```

### 5.3 "착취율" → "헤게모니 수수료(Hegemonic Fee)"

```
헤게모니_수수료율 = 누적 약세율 (사후 확정치)

KRW 2026: 23.8% | JPY 2026: 26.6% | TWD 2026: 9.2%

실질_수취액 = 명목흑자 × (1 / (1 + 헤게모니_수수료율))

KRW: $160B → $129B (수수료 $31B)
JPY: $240B → $190B (수수료 $50B)
TWD: $250B → $229B (수수료 $21B)
```

### 7.3 [v3.2 신규] 표본 범위 논쟁 — "왜 10개국 다 안 보냐"는 지적에 대한 정면 반박

외부 3차 평가(Perplexity)는 "KRW/JPY/TWD 3국 패턴이 독일·스위스·싱가포르까지 일반화되는지 검증 필요"라고 지적했다. **이 지적은 학술논문 기준으로는 맞지만, 이 백서의 실제 용도 기준으로는 틀렸다.** 이유를 명확히 한다.

**두 가지 다른 목적을 혼동하면 안 된다:**

```
목적 A: "달러 패권 가설이 전 세계 보편적으로 성립하는가"
        → 학술 일반화 검증. 10개국 전부, 가능하면 전체 무역상대국 필요.
        → 이건 Hudson/Varoufakis가 하려던 작업. 이 백서의 목적이 아니다.

목적 B: "내(박씨)가 실제로 노출되어 있는 통화 리스크를 관리한다"
        → 박씨의 실제 경제활동(원화 소득, 한국 거주, 콘텐츠 사업)이
          물리고 있는 통화는 KRW가 본진이고, 사업/콘텐츠 연관성으로
          JPY(엔화 콘텐츠 시장)·TWD(반도체 공급망 비교축)가 참조축이다.
        → 독일(EUR), 스위스(CHF), 싱가포르(SGD)는 박씨의 실제 현금흐름과
          무관한 통화. 검증해도 박씨의 의사결정에 아무 영향 없음.
```

**이 백서는 목적 B를 위해 설계됐다.** 목적 A의 일반화 요구는 카테고리 오류(category error)다. 의사 결정 모델에 학술 검증 기준을 들이대는 것 — 마치 "내 가족 가계부가 OECD 전체 가구의 소비패턴을 대표하는지 증명하라"는 요구와 같다. 가계부는 일반화를 목적으로 만들지 않는다.

### 7.4 [v3.2 핵심] 변수 축소 원칙 — 다변량을 2개 유닛으로 환원

박씨의 반박 핵심: "저 변수들을 모델링해서 금 또는 USD로 환원시키는 두 가지만 유닛화하면 지적사항 자체가 사라지는 거 아니냐."

**정확하다. 이것이 v3.2의 핵심 설계원칙이 된다.**

```
[Perplexity가 원하는 구조]          [박씨가 실제로 필요한 구조]
─────────────────────────         ─────────────────────────
KRW, JPY, TWD, EUR, CHF,           KRW_exposure ──┐
SGD, CNY, VND... (N개 변수          JPY_exposure ──┼─→ [통합 환산기] → GOLD_unit
각각 따로 추적+검증)                TWD_exposure ──┘         또는
                                                          USD_unit

문제점: N이 커질수록                장점: N이 몇 개든 최종 출력은
- 연산비용 폭증 (8장에서 이미        2개 유닛(금/달러)으로 항상 수렴
  박씨가 지적한 문제)               → 검증 대상이 "환산 로직 1개"로
- 검증해야 할 데이터소스도 N배         고정되어 오히려 검증이 쉬워짐
- 정작 의사결정에 안 쓰임
```

**왜 "검증이 쉬워지는가"가 핵심인가**: Perplexity 지적의 숨은 전제는 "변수가 많을수록 모델이 부실하다"는 것이다. 그러나 실제로는 반대다. **모든 입력변수가 동일한 최종 유닛(금 또는 USD)으로 강제 환산되면, 검증해야 할 지점이 "각 통화의 개별 행동"이 아니라 "환산 함수 하나"로 줄어든다.** 이건 회계학의 연결재무제표(consolidated statement) 원리와 동일 — 자회사가 몇 개든 모두 지배통화로 환산되면 감사 포인트는 환산율 하나로 수렴한다.

### 7.5 2-유닛 환원 모델 (Dual-Unit Reduction Model)

```
입력 다변량 (N개, N은 박씨의 실제 노출국에 따라 가변):
  KRW_exposure(t), JPY_exposure(t), TWD_exposure(t), [...필요시 추가]

환원 함수 1 — GOLD 유닛화:
  GOLD_equiv(t) = Σ[i] Exposure_i(t) × FX_to_USD_i(t) / GOLD_price_USD(t)

환원 함수 2 — USD 유닛화:
  USD_equiv(t) = Σ[i] Exposure_i(t) × FX_to_USD_i(t)

최종 의사결정 변수는 이 2개뿐:
  GOLD_equiv(t), USD_equiv(t)

→ 13장 3-Bucket 모델의 입력값이 정확히 이 2개 유닛이었다.
  즉 v3.0 설계 당시 이미 이 원칙이 암묵적으로 적용되어 있었고,
  v3.2에서 그 원칙을 명시적 정식 모델로 승격한 것.
```

**이 환원이 7.3절 표본범위 논쟁을 어떻게 해소하는가:**

```
독일/스위스/싱가포르를 추가로 검증해야 하는 경우는 단 하나:
  → 박씨가 EUR/CHF/SGD 표시 자산이나 소득을 실제로 보유하게 될 때

그 전까지는:
  → N=3(KRW/JPY/TWD)이든 N=10(재무부 전체리스트)이든
    최종 출력은 어차피 GOLD_equiv, USD_equiv 2개로 수렴
  → 안 쓰는 변수를 늘려봐야 의사결정 정밀도에 기여 0
  → 오히려 GATE 0(14장) 통과 대상만 늘어나 연산비용·오염위험만 증가
```

### 7.6 학술모델과 실행모델의 경계선 — 백서의 정체성 재확인

이 절은 백서 전체의 정체성을 다시 한번 명확히 한다.

```
이 백서 = "Hudson/Varoufakis식 보편이론"이 아니라
        "박씨 개인/사업체의 통화노출 관리를 위한 실행모델"

학술 보편이론이 되려면: 10개국+α 전체 검증 필요 (Perplexity 말이 맞음)
실행모델로 충분하려면: 실제 노출국(KRW/JPY/TWD) + 2-유닛 환원이면 충분 (박씨 말이 맞음)

이 백서는 후자를 선언한다. 전자를 원한다면 별도의 학술판(academic edition)이
필요하며, 이는 이 문서의 스코프 밖이다.
```

> **[v3.2 주의]** 이 결정이 "Perplexity가 틀렸다"는 뜻은 아니다. 다른 목적(학술발표, 정책제안)으로 이 백서를 쓴다면 Perplexity 지적이 정확히 맞다. 그러나 이 백서의 1차 목적(13장 개인헤지전략 + 15장 MCP)에서는 그 작업이 불필요한 과잉공학(over-engineering)이다. 목적에 따라 다른 기준이 적용된다는 것을 명시하는 것 자체가 학술적 정직성이다.

```

이것만으로는 트레이딩에 쓰기엔 약하다. **→ 12장에서 버지팅 모델로 업그레이드.**

## 제12장: 버지팅 기반 환율 선행 모델 (Rolling Forecast Engine)

### 12.1 박씨의 직관 — 예산 버지팅 구조 그대로 이식

> "Q3 정도에서 실적가 나오고 마지막 분기는 예상치를 집어넣고, 그 예상치를 투자은행/신용평가기관/세리 연구소 자료에 가중치 주고, 업종별로 미국에서 중요하다는 데에 또 가중치를 주면..."

이건 정확히 기업 **Rolling Forecast(연동예측)** 구조다. 회계실무에서 매 분기 쓰는 기법을 환율에 그대로 이식 가능하다.

### 12.2 구조 매핑

```
[기업 버지팅 모델]                    [환율 선행 모델]
────────────────────                ────────────────────
Q1~Q3 실적 확정치                  → 1~3분기 실제 환율/경상수지 확정치
Q4 예상치 (관리자 추정)            → 4분기 Fed 금리경로 + 경상수지 컨센서스
IB/신용평가 리포트 가중치          → 주요 IB(GS/JPM/MS) FX 전망 가중평균
업종별 중요도 가중치               → 미국 입장 전략업종(반도체/자동차) 노출도 가중
최종 예산 = Σ(확정치+가중예상치)   → 최종 환율전망 = Σ(확정치+가중예상치)
```

### 12.3 연산 식

```
FX_forecast(연말) = 
    Σ[t=1..3] Actual(t) × w_actual
  + Forecast(Q4) × w_estimate

Forecast(Q4) = Σ[i=1..n] IB_estimate(i) × Credibility_weight(i) × Sector_weight

여기서:
- Credibility_weight(i) = IB(i)의 과거 4개분기 적중률 기반 가중치
- Sector_weight = 해당국 수출에서 미국 전략업종(반도체/자동차/배터리) 비중
- w_actual, w_estimate = 분기 진행률에 따른 동적 가중치 (Q3말이면 actual 75%, estimate 25%)
```

### 12.4 입력 데이터 소스 (전부 공개 데이터)

| 데이터 | 출처 | 갱신주기 |
|--------|------|---------|
| 분기 경상수지 확정치 | 한국은행/BOJ/CBC 공식발표 | 분기 |
| Fed 금리경로 컨센서스 | CME FedWatch | 실시간 |
| IB 환율전망 | GS/JPM/MS/Citi 분기 FX 아웃룩 (공개 요약본) | 분기 |
| 신용평가 전망 | Moody's/S&P 국가등급 코멘터리 | 비정기 |
| 업종 수출비중 | 한국무역협회/관세청 | 월간 |
| 재무부 모니터링 리스트 | US Treasury FX Report | 반기 |

### 12.5 신뢰도 가중치 설계 (SCM 모델 적중률 로직 재사용)

```python
# 개념 코드 — 실제 구현은 14장 MCP에서
credibility_weight = (
    0.4 * IB_historical_accuracy(lookback=4_quarters) +
    0.3 * IB_market_share_in_FX_research +
    0.3 * (1 - IB_forecast_dispersion)  # 전망 분산 작을수록 신뢰
)
```

### 12.6 한계와 정직한 평가

이 모델은 **9.2절의 "선행지표 불가" 비판을 부분적으로만 해결**한다.

- IB 컨센서스 자체가 이미 "시장의 사후 학습"을 반영한 것 → 완전한 독립 선행지표는 아님
- 그러나 **"공개된 여러 추정치를 박씨식 가중치(헤게모니 수수료 모델)로 재필터링"**하는 메타모델로서는 충분히 차별성 있음
- 백테스트 없이 신뢰 불가 → **MCP 1차 목표는 백테스트 엔진** (15장)

## 제13장: 개인 투자자를 위한 비-FX 헤지 전략

### 13.1 박씨의 직관

> "FX 시장에서 거래 자체를 하지 않으니까, 금 같은 절대 매개를 하나 정해서 매집 기간을 설정하자. 원화 가치 좋을 때 USD를 사자."

**핵심 통찰**: 개인은 인터뱅크 카르텔(8장)도, Fed 정책 결정(4장)도 손댈 수 없다. 그러나 **"헤게모니 수수료를 거꾸로 이용하는 타이밍 전략"**은 가능하다.

### 13.2 전략 명제

```
경상흑자기 + 원화 일시 강세 구간 = "형이 나중에 뺏어갈 돈"이 잠깐 내 손에 있는 시점
→ 이때 달러 표시 자산(USD 현금, 금)으로 환전해 둠
→ 다음 사이클(Fed 긴축/원화 약세) 때 그 자산이 상대적으로 방어됨
```

이건 트레이딩이 아니라 **DCA(Dollar-Cost Averaging) 기반 환헤지**다. 환투기가 아니라 자산배분.

### 13.3 매집 시그널 설계 — 12장 모델과 연결

```
매집_시그널 = 1, if:
  (원화가 12장 FX_forecast 대비 일시 고평가 — "원화 강세 윈도우")
  AND (재무부 모니터링 리스트 등재 압박 임박 — 7장 패턴)
  AND (Fed 긴축 사이클 초입 — 9.3절 FOMC 신호)

매집_금액 = 월 고정액 × (1 + 시그널_강도)
```

### 13.4 구체적 자산 배분 모델 (3-Bucket)

| 버킷 | 자산 | 역할 | 비중(예시) |
|------|------|------|-----------|
| A | USD 현금/MMF | 즉시 유동성 헤지 | 40% |
| B | 금(KRX금시장/실물) | 달러 자체 리스크 헤지 (탈달러 보험) | 35% |
| C | 달러표시 우량채(미국채단기) | 이자수익 + 환헤지 동시 | 25% |

**왜 금을 같이 넣나**: 박씨 모델의 결론(4장)은 "미국이 판의 기울기를 정한다"는 것. 그런데 달러 자체도 미국 부채 시스템 안에 있는 자산이다. 순수 달러 올인은 "패권국 화폐를 또 다른 패권국 화폐로만 헤지"하는 모순. **금은 어느 나라 부채도 아닌 유일한 비-소버린 자산**이라 구조적으로 다른 층위의 헤지가 된다.

> **[v3.2 연결]** 이 3-Bucket 모델의 입력값은 7.5절 Dual-Unit Reduction Model의 출력인 `GOLD_equiv(t)`와 `USD_equiv(t)` 두 변수다. 박씨가 KRW든 JPY든 TWD든 몇 개 통화에 노출되어 있든 상관없이, 이 단계에서는 이미 2개 유닛으로 환원이 끝난 상태로 들어온다. **표본국가 수(N)는 7장에서 끝나고, 13장은 N과 무관하게 항상 동일한 2변수 모델로 작동한다.** 이것이 7.6절에서 선언한 "실행모델"의 실제 작동 방식.

### 13.5 실행 규칙 (개미 투자자용 — 심플 버전)

```
규칙 1: 매월 고정액 적립 (시그널 무시 베이스라인) — 행동편향 방지
규칙 2: 12장 시그널 강도가 임계치 넘으면 추가 매집 (스킵 아님, 증액만)
규칙 3: 절대 레버리지 금지 — 이건 헤지지 투기가 아님
규칙 4: 리밸런싱은 연 1회만 — 단기 노이즈에 반응 금지
```

### 13.6 이 전략의 정직한 한계

- 이건 알파(초과수익) 전략이 아니라 **베타 헤지(구조적 손실 방어) 전략**이다
- 9장에서 명시했듯 단기 타이밍은 예측 불가 — 그래서 DCA가 핵심이지 마켓타이밍이 아님
- 세금/환전수수료 등 실무 비용은 별도 고려 필요 (이 백서 범위 밖)

## 제14장: 0번 게이트 — 로데이터 역검증 엔진 (Data Provenance & Integrity Gate)

### 14.1 문제 제기 — Gemini 동료평가 + 박씨 역제안

외부 2차 평가(Gemini)는 "재무부/IB 데이터가 갑자기 기준을 바꾸거나 산출방식이 변경될 때 시스템이 튀지 않도록 Health Check이 필요하다"고 지적했다. 박씨는 여기서 한 발 더 나갔다:

> "저런 IB나 오픈소스로 풀리는 통계청·미국 자료가 의도를 가지고 선동하거나 로데이터를 조작하는 거를 거꾸로 추적하고 검증하고, 내가 그걸 역용할 수 있는 플러그가 첫 번째 게이트에 있어야 될 것 같은데."

이건 단순 장애감지(Health Check)가 아니라 **데이터 자체의 의도성을 검증하는 역추적 게이트**다. 백서의 전체 논리(4장: 미국이 판의 기울기를 세팅한다)를 그대로 데이터 신뢰성 검증에도 적용해야 일관성이 생긴다 — **"환율도 조작 가능하면, 환율을 만드는 원천 통계도 조작 가능하다"**는 것이 같은 명제의 연장선이기 때문.

### 14.2 왜 이게 단순 Health Check보다 중요한가

```
Health Check (Gemini 제안) = "데이터가 안 들어오면 멈춘다" → 가용성 문제
Provenance Gate (박씨 역제안) = "데이터가 들어와도 의심한다" → 신뢰성 문제
```

가용성은 쉽다. 데이터가 없으면 에러 띄우면 끝. 진짜 위험은 **데이터가 멀쩡하게 들어오는데 그 안에 의도가 섞여있는 경우**다. 이건 백서 8장(FX 카르텔)에서 이미 확인한 패턴과 구조적으로 동일하다 — "시장이 작동하는 것처럼 보이지만 안에 공모가 있다"는 명제를 **데이터 공급망에도 적용**하는 것.

### 14.3 0번 게이트 아키텍처 — 모든 모듈 진입 전 강제 통과

```
┌──────────────────────────────────────────────────┐
│  [GATE 0: Provenance & Integrity Check]            │
│  모든 외부 로데이터는 이 게이트를 통과해야만           │
│  Module 1~8로 전달 가능 (우회 불가, 하드코딩)          │
└──────────────────────────────────────────────────┘
                      ↓
        [통과] → Layer 2(로데이터 수집)로 정상 진행
        [경고] → 사용자에게 의심신호 명시 + 보수적 가중치 자동 하향
        [차단] → 해당 데이터소스 일시 배제 + 대체소스 탐색
```

### 14.4 게이트 0의 5단계 검증 로직

**Step 1: 출처 계보 확인 (Source Chain Verification)**
```
- 1차 출처(한국은행/Fed 원문)인지, 2차 가공(기사/블로그 인용)인지 구분
- web_fetch한 URL이 .gov, 중앙은행 공식도메인인지 vs 민간 집계사이트인지 태깅
- 2차 출처는 자동으로 신뢰도 -30% 페널티
```

**Step 2: Benford's Law 1차 분계기**

통계 데이터(GDP, 무역수지, 경상수지 등 집계수치)에서, 자연발생적으로 발생하는 숫자 데이터셋은 작은 숫자(특히 '1')가 첫자리에 더 자주 등장하는 패턴이 있고, 경제학자/금융분석가들이 GDP·주가·물가지표 같은 거시통계를 이 법칙으로 검증한다.

```python
# 개념 설계 — Module 14에서 실제 구현
def benford_check(data_series):
    first_digits = extract_first_digit(data_series)
    observed_dist = compute_distribution(first_digits)
    expected_dist = benford_theoretical_distribution()
    deviation = chi_square_test(observed_dist, expected_dist)
    return deviation  # 임계치 초과 시 GATE 0 경고 플래그
```

> **한계 정직히 명시**: Benford 법칙은 항상 모든 케이스에 작동하는 건 아니고, 작은 숫자로의 자연적 편향이 워낙 많은 실세계 데이터셋에서 발견되는 패턴이라 무언가 있다고 단정하기엔 부적절한 경우도 있다. 실제 연구에서도 Benford 법칙은 그 자체로 완벽한 도구는 아니며, 깃발이 꽂힌 건마다 추가 조사가 필요하다는 한계가 있다. → **GATE 0은 Benford 이탈을 "확정 조작 증거"가 아니라 "추가 검증 트리거"로만 사용한다.**

**Step 3: 시계열 불연속성 탐지 (Regime Change Detector)**
```
- 같은 지표(예: 재무부 FX Report 등재기준)가 갑자기 임계값을 바꾸는지 추적
- 과거 N개 보고서의 기준선과 현재 보고서 기준선 자동 diff
- 기준 변경 발견 시 → "왜 바뀌었는가"를 별도 web_search로 자동 조사 후 사용자에게 보고
```

**Step 4: 발행기관 인센티브 분석 (Issuer Incentive Flag)**
```
이 데이터를 발표한 기관이 그 수치로 이득을 보는 위치인가?

예시:
- 재무부가 "한국 환율조작 아님" 판정 → 무역협상 직전이면 정치적 인센티브 의심 태그
- IB가 "내년 원화 강세 전망" → 그 IB가 KRW 표시 자산을 대량 보유중이면 이해상충 태그
- 신용평가사 등급변경 직전/직후 환율급변 → 선행정보 유출 의심 태그
```

**Step 5: 교차검증 합의도 (Cross-Source Consensus Score)**
```
동일 지표를 3개 이상 독립 출처(정부공식/IB/제3국 기관)에서 동시 수집
→ 표준편차가 임계치 넘으면 "출처간 불일치" 경고
→ 불일치 클수록 GATE 0 신뢰도 점수 하락 → Module 6(Rolling Forecast) 가중치 자동 축소
```

### 14.5 박씨의 "역용" 아이디어 — 역추적을 공격이 아니라 신호로 전환

박씨가 말한 "역용할 수 있는 플러그"의 핵심은: **조작 시도 자체가 하나의 정보(시그널)라는 역발상.**

```
명제: 누군가 데이터를 조작하려는 시도가 보인다면,
     그 자체가 "이 지표가 지금 중요해졌다"는 역설적 신호다.

활용 로직:
GATE 0이 특정 지표에서 반복적으로 이상신호를 잡아낸다면
→ 그 지표를 Module 8(매집 시그널)의 "주의 깊게 볼 변수" 리스트에 자동 승격
→ "누군가 이 숫자를 관리하고 싶어한다 = 이 숫자가 다음 정책결정의 핵심이다"
```

이건 8장의 "가마우지" 모델과 정확히 같은 구조다. 카르텔이 Fix를 조작한다는 것 자체가 그 60초 윈도우가 돈이 된다는 증거였듯, **특정 통계가 자꾸 도마에 오른다는 것 자체가 그 통계의 정치적 비중이 커지고 있다는 선행신호**로 역이용 가능.

### 14.7 GATE 0이 절대 하면 안 되는 것

❌ 역용 시그널을 트레이딩 확정신호로 직결 — 14.5는 가설적 휴리스틱일 뿐
❌ 게이트 자체를 우회하는 모듈 허용 — 모든 외부데이터는 예외없이 통과 필수
```

## 제15장: MCP 풀스펙 — LLM 로데이터 수집 → Python 강제실행 → 투자모형 래퍼

### 15.1 설계 철학

이 백서는 설명모델이지만, MCP는 **"매번 사람이 손으로 데이터 긁어다 엑셀 만드는 일"**을 자동화하는 게 핵심 가치다. LLM이 자연어로 떠들지 않고 **반드시 Python 코드를 실행해서 숫자를 내야** 신뢰 가능한 도구가 된다.

### 15.2 전체 아키텍처

```
┌─────────────────────────────────────────────────┐

### 15.3 "Python 강제" 설계가 왜 필수인가

LLM이 자연어로 "약 24% 정도 약세였습니다"라고 말하면 **숫자가 매번 미묘하게 달라지고 검증 불가**하다. 이걸 막는 게 핵심 가드레일:

```
원칙: LLM은 절대 최종 숫자를 "기억"이나 "추정"으로 답하지 않는다.
     반드시 Python 스크립트를 실행한 stdout 결과만 인용한다.

구현:
1. LLM이 계산 요청을 받으면 → 자연어 답변 금지
2. create_file로 .py 스크립트 작성 (12.3절 수식 그대로 구현)
3. bash_tool로 실행
4. stdout/CSV 결과만 사용자에게 전달
5. 코드 자체도 함께 공개 (재현가능성 = 신뢰성)
```

  - 이 모듈이 "표본 일반화 논쟁(7.3절)"을 구조적으로 무력화하는 지점:

    하위 모듈은 전혀 영향받지 않음 (느슨한 결합 / loose coupling 설계)
