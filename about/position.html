02

PWA에서 웹페이지로 — 서비스 워커 제거 결정

서비스 워커 "기능" 실제 문제
오프라인 캐시코드 수정해도 옛날 버전 표시
캐시 전략크롬 번역 기능 먹통
PWA 쉘브라우저 TTS 읽어주기 안 됨
앱 격리오버레이/방송킷 기능 제한
웹페이지 = 칠판 (방송 화면) 화면 녹화 = 촬영 녹화본 = 완성물 (편집 0분) → YouTube 업로드

서비스 워커 제거 후: git push → 즉시 반영 · 크롬 번역·TTS 정상 · 오버레이 방송킷 정상.

교과서가 아니라 자기 용도에 맞는 판단. 제약이 설계를 결정한다.

08

결론 — 자기참조 생산 체계

이 생태계의 원재료는 대화다.

기능인 박씨가 12시간 식당 노동 후 핸드폰 Termux에서 자연어로 설계하는 그 대화 자체가 원유이며, OrbitPrompt는 그 원유를 정제하는 정유소다.

3개월 만에 도달한 것은 단순한 레포 28개가 아니라, 자기 자신을 입력으로 받아 자기 자신을 개선하는 자기참조 생산 체계다.

OrbitPrompt는 28개 레포의 심장이자 두뇌다.

공장을 만드는 공장. 생각을 만드는 생각. 프롬프트를 만드는 프롬프트.
