# 인체 · IT · 국가 시스템 매핑

> 하나의 시스템 언어로 정렬하는 작업.
> 의학 · CS · 국가 시스템을 같은 패턴으로 보면 세 분야가 동시에 이해된다.

---

## 개요

의학에서 인체를 **여러 시스템(신경·면역·순환·내분비…)**으로 나누고,
CS에서 컴퓨터/네트워크를 **레이어·컴포넌트·프로토콜·보안·오케스트레이션**으로 보고,
국가 시스템을 **부처·인프라·재정·안보·복지·외교**로 보는 것을,
전부 **"시스템 구성요소 + 신호 + 자원 + 피드백"**이라는 한 언어로 다시 묶는다.

이건 돌려맞추기가 아니라 **기능·역할 기준 추상화**다.
각 분야의 공식 구조와 역할을 먼저 보고, 역할 레벨에서만 매칭하는 것.
같은 시스템 패턴을 쓰면 한 분야에서 배운 직관이 다른 분야로 곧장 전이된다.

## 핵심 4축

| 축 | 인체 | IT | 국가 | 예시 |
|----|------|----|------|------|
| **제어** | 신경계 | CPU·OS·제어 플레인 | 대통령·국회·정책 결정 | 재발 헤르페스 ↔ 사이버 공격 ↔ 국가 위기 |
| **수송** | 순환계 | 네트워크·데이터버스 | 재정·물류·에너지 | 혈압·당뇨 ↔ 네트워크 혼잡 ↔ 물가·금리 |
| **방어** | 면역계 | 보안·헬스체크 | 국방·경찰·방역 | 염증 반응 ↔ DDoS 대응 ↔ 재난 안전 |
| **조율** | 내분비계 | 설정·스케줄러 | 재정·복지 정책 | 당뇨·갑상선 ↔ QoS·컨피그 ↔ 세율·금리 |

## 이 매핑이 주는 이점

1. **전이 가능한 직관** — 한 도메인에서 배운 직관이 다른 도메인으로 바로 이전된다
2. **모듈식 사고** — 각 시스템의 역할을 알면, 다른 도메인의 낯선 개념도 역할 기준으로 이해 가능
3. **공통 언어** — 의학·CS·국가 시스템을 하나의 시스템 언어로 논의할 수 있다
4. **효율적 학습** — 따로따로 외우지 않고 공통 패턴으로 압축해서 머릿속에 하나의 구조물로 올린다
