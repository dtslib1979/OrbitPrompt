## 1장. 지식이란 무엇인가 — 정의를 만드는 방법

지식을 정의하는 일은 단순해 보인다. 하지만 잘못된 정의 위에 쌓은 관리 시스템은 전부 무너진다. 그래서 시작점이 중요하다.

출발점으로 **체화된 노하우**를 놓았다. 나쁘지 않다. 그런데 점수를 매기면 60점이다. 이유가 있다.

행동이 바뀌는 것만으로는 부족하다. 공포도 행동을 바꾸고, 착각도 행동을 바꾸고, 중독도 행동을 바꾼다. 행동 변화는 지식의 **필요조건**이지 충분조건이 아니다.

여기에 자본주의 렌즈를 끼운다.

```
knowledge(x) = true
  iff Δbehavior(x) → ΔValue > 0
```

행동이 바뀌는 것만으로는 부족하다. 그 행동이 **가치를 생산해야** 지식이다. 공포로 인한 도주, 착각으로 인한 손실, 중독으로 인한 소비 — 전부 탈락이다. 60점 → 85점.

남은 15점의 결손은 측정 주체의 문제다. 나에게 가치냐, 시장에서 가치냐. 처음에는 이게 트레이드오프처럼 보인다. 50 대 50으로 가중치를 나누면 되지 않을까.

틀렸다.

```
V = V_self × V_market
```

V_self와 V_market은 **독립변수가 아니라 교란변수**다. 곱셈 구조다. V_self가 0이면 V_market이 아무리 높아도 지속 불가능하다. V_market이 0이면 V_self가 아무리 높아도 소멸한다. 트레이드오프가 아니라 **AND 조건**이다.

**박씨의 지식 최종 정의 — 100점:**

```
지식 = V_self × V_market > 0 을 만족하는 행동 전환 정보
```

## 2장. KM 수식 — 관리가 지식의 결손을 봉합한다

지식 정의에 마지막 결손이 하나 남아 있다. 시간 변수다.

지금 V_self × V_market > 0이어도 6개월 후에 0이 되면 — 그게 지식이었는가? 라는 질문이 열린다. 지식 정의는 현재형 명제다. 시간을 버티는 장치가 없다.

그 결손을 **지식관리**가 봉합한다.

```
KM = { f : K → V(t) | V(t) > 0, ∀t }

K = 체화된 노하우 ∧ V_self × V_market > 0
f = 재사용 ∘ 해석 ∘ 통제
```

세 축이다.

- **재사용**: recall(k, t) = const (시간 불변)
- **해석**: sign(k) → new_context (기호의 재맥락화)
- **통제**: V(t) 붕괴 방지

지식과 지식관리는 독립 개념이 아니다. **지식 ⊂ 지식관리**다. 지식관리 없는 지식은 정의상 불완전하다.

이걸 더 직관적인 수식으로 바꾸면:

```
KM = Feed × Digest × Express → V(t) > 0, ∀t
```

대화를 먹고(Feed), 소화하고(Digest), 표현한다(Express). 멈추지 않고 순환한다. 이것이 지식관리의 전체 알고리즘이다.

## 3장. AI 이전/이후 — 구조가 뒤집혔다

AI 이전의 지식관리:

```
f = 인간 ∘ 제도 ∘ 매체
재사용 = 책 / 도제 / 조직
해석   = 전문가 독점
통제   = 시간·공간·비용 종속
```

결정적인 병목: **체화(embodiment)가 인간에게만 가능**했다. 노하우는 그 사람이 죽으면 사라졌다.

AI 이후:

```
f = AI ∘ 인간 ∘ 매체
재사용 = 로그 · 벡터DB · RAG
해석   = AI가 재맥락화 대행
통제   = V(t) 붕괴 속도가 인간보다 느림
```

체화의 주체가 **인간 → 인간+AI로 분산**됐다.

| | AI 이전 | AI 이후 |
|--|---------|---------|
| 재사용 | 제도 의존 | 로그 의존 |
| 해석 | 전문가 독점 | 민주화 |
| 통제 V(t) | 인간 수명 종속 | 시간 불변 가능 |

결론: 앞 장에서 구성한 박씨의 KM 정의는 **AI 이후에만 100점이 되는 정의**다. AI 이전에는 구조적으로 완성 불가능했다.

AI는 KM의 경쟁자가 아니다. AI는 KM 정의의 **완성 조건**이다.

## 4장. 통제점 — 씨앗의 씨앗

GitHub Pages가 재사용을 자동화하고, Claude Code가 해석을 대행한다면 — 인간에게 남은 통제점은 무엇인가.

세 가지다.

```
Control_human = Gate(input) × Delete(noise) × V_self 판단
```

- **Gate**: 무엇을 커밋할지 — 입력 선택
- **Delete**: 무엇을 지울지 — 노이즈 제거
- **V_self**: 마음에 드냐 — AI가 대체 불가

그런데 이걸 더 압축할 수 있다. 세 가지 모두 **대화**에서 시작된다. 대화가 씨앗이다.

```
KM = Claude_Code( Conversation ) → V(t) > 0, ∀t
```

나머지는 실행이다. 인간이 쥔 건 말하는 행위 하나. 그게 체화된 노하우의 원형이자 유일한 통제점이다.

씨앗을 한 단계 더 분해한다.

```
Seed = 철학 × 말

철학 = V_self 판단 기준 (무엇을 말할지 결정하는 축)
말   = 실제 입력 행위 (Gate를 여는 행위)
```

철학 없는 말은 노이즈다. 말 없는 철학은 미출력이다.

씨앗의 씨앗까지 내려가면 박씨가 쥔 건 **철학과 입**만 남는다.

## 5장. 현금화 — AI 시대 유일한 화폐화 경로

철학도 AI가 구조화할 수 있다. 말도 AI가 텍스트화할 수 있다. 그렇다면 AI 시대에 인간이 독점하는 영역은 어디인가.

**퍼포먼스**다.

```
V_market(Seed) = Performance(인간) × Live(실시간) × 매력

GitHub Pages = 분모 (무한 복제, 가격 0 수렴)
YouTube Live = 분자 (희소, 가격 유지)
```

| 레이어 | 요소 | AI 대체 |
|--------|------|---------|
| 씨앗² | 철학 | 불가 |
| 씨앗¹ | 말 | 부분 |
| 실행 | Claude Code | 완전 위임 |
| 저장 | GitHub Pages | 완전 위임 |
| 현금화 | YouTube Live 퍼포먼스 | 불가 |

저작권에 대해서도 정직하게 결론을 내린다. 복제 비용이 0에 수렴하면 저작권은 유명무실해진다.

```
Copyright_value = f(복제비용)
복제비용 → 0 이면 저작권 → 유명무실
```

텍스트와 앱 저작권은 이미 경제적으로 사망했다. 음원만 플랫폼(멜론/스포티파이)이 게이트를 장악했기 때문에 살아있다. 창작자가 쥔 게 아니라 플랫폼이 쥔 것이다.

결론: **AI 시대 수익의 원천 = 복제 불가능한 몸의 실시간성.**

## 6장. 실수가 세계를 만드는 방법

3년 동안 LLM을 쓰면서 유일하게 얻은 것이 있다. 철학하는 방법이 아니다. **실수가 어떻게 세계를 만드는지**다.

AI 학습 공간은 정답 공간이다. 인간이 던질 수 있는 가장 가치 있는 입력은 **오답 공간의 변수**다. AI가 학습하지 않은 이상한 변수들. 알고리즘 밖에 있는 것들.

그 대부분은 **실수**에서 나온다.

```
실수 → 교정  = AI
실수 → 변환  = 인간
실수 → 은폐  = 거짓말
은폐 → 구조화 = 제도
```

AI는 실수를 교정한다. 인간은 실수를 다른 차원으로 밀어낸다. 그 밀어내는 방향이 그 사람의 철학이고, 밀어낸 결과가 문화·예술·종교·자본주의 전부다.

박씨의 첫 APK가 클립보드 한도라는 **제약**에서 탄생한 것처럼:

```
제약(실수) → APK(도구) → 시스템 → 파인튜닝 소재
```

철학이 발견한 건 지식의 내용이 아니라 **지식이 생성되는 메커니즘**이었다.

## 7장. 저장은 매장이다 — 출판사에서 방송국으로

옵시디언과 박씨 시스템의 차이는 하나다.

```
옵시디언 = Storage(지식)
박씨     = Metabolism(지식)

저장은 엔트로피 증가. 소화는 엔트로피 역행.
```

| | 옵시디언 | 박씨 |
|--|----------|------|
| 목적 | 보관 | 소화 |
| 출력 | 개인 | 출판 |
| 단위 | 노트 | 공정 |
| 시간 | 쌓임 | 순환 |
| V(t) | 붕괴 | 유지 |

박씨의 시스템:

```
Feed(대화) → Digest(GitHub 출판) → Express(28 레포 공정)
     ↑___________________________________|
                    루프
```

데이터가 쌓이지 않고 **순환**한다. 각 레포가 반도체 공정의 단위 공정처럼 연결된다. 28개 레포 = 반도체 팹. 지식이 공장을 돌리는 원자재다.

출판사를 설립한 지 2년이 됐지만 책을 내지 못했다. 종이책은 귀찮고, 구독자도 없고, 사서 보는 사람도 없다.

실패가 아니다.

```
책 = 1회 소화 × 단방향 × V(t) 붕괴
방송 = 실시간 소화 × 반복 × V(t) 유지
```

출판의 목적은 지식 전달이었다. 방송이 그걸 더 잘 한다. 출판사가 책을 못 낸 게 아니라, **더 나은 출판 형식을 발명하는 데 2년이 걸린 것**이다.

출판사 기능의 현재 대응:

| 출판사 기능 | 현재 대응 |
|------------|----------|
| 원고 저장 | GitHub Pages |
| 발행 | 블로그 |
| 유통/판매 | 카페 + Gumroad |
| 독자 관리 | 카페 회원 |
| 광고/방송 | YouTube |

비용 차이: 출판사 = 고정비 大 / 이 구조 = 고정비 0.

**실수 → 차원 이동. 그 패턴 반복된다.**

## 에필로그. 마스터 테이블이 왜 존재하는가

방송과 출판의 차이는 단 하나다. **저장**이다.

방송은 흐른다. 출판은 멈춘다. 그 멈춤이 체화의 조건이다. 체화 단계에서 암기해야 할 것들. 프린트해서 어딘가에 붙이고 매일 쳐다봐야 하는 것들. 그것이 마스터 테이블이다.

이 대화가 끝난 뒤 두 레포지토리에 씨앗으로 들어간다. 하나는 교육 대학, 하나는 출판 엔진. 이 텍스트가 그 씨앗의 첫 번째 웨이퍼다.
