PHL Protocol

상태 전이 인터페이스

PHL(Protocol for Human Language) — 짧은 토큰이 엄격한 규칙을 실행한다. 단축키가 아니라 상태 전이(state transition) 인터페이스다.

PHL-1
SPEC 로드 → 상태 점검
PHL_SPEC.md + PHL_INDEX.json 로드. 현재 파일 구조/테스트/의존성 확인. 계획 보고 → 실행 → 검증 → 커밋 사이클.
PHL-2
Dual Compiler
Lane A (University): 사건→지식 컴파일. Lane B (Orbit): 대화→프롬프트 컴파일. 두 레인 동시 실행.
PHL-3
표준 출력 형식
Loaded Specs → Plan → Changes → Validation → Commit → Notes. 이 형식에서 벗어난 보고 없음.
