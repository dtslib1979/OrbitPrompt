## 3. 4축별 결합 구조

### 🚀 a축: 경제적 자립 (최우선)

```
문제 찾기 → AI 리서치
  → 결과물 설계 → AI 기획 보조
    → 실행물 제작 → 글·영상·앱 자동 생성
      → 배포·판매 → 유튜브 이상 다채널 운영
        → 수익 검증 → 실제 고객 반응과 결제 경험
```

**핵심:** 자립은 이론이 아니라 "AI를 부려 매출 또는 실질 가치를 만들 수 있는가"로 측정된다.

**연결점:** 오늘 만들어진 MCP 7개가 이미 이 파이프라인의 기술적 기반이다.

### 🧠 h축: 인간성·돌봄 (가드레일)

돈을 버는 능력만으로는 인간을 지킬 수 없다. 약자·욕망·죄책감·책임·관계에 대한 언어가 필요하다.

**플러그-인 내부 장치:**
- 철학·종교·의미 세션 — 과정 내 필수 모듈
- 실패 감정 처리 — 자기책임론으로 귀결되지 않도록
- 공동체 규범 — AI가 약자 짓밟는 도구가 되지 않게

**연결점:** Media MCP Flux 슬롯, 브랜치 철학, HUMILITY-CHECK.md가 이미 이 구조를 포함한다.

### 👥 s축: 선택 기반 사회화

기존 사회화(학교·회사)는 강제 공동체 중심이다. 네트워크 시대에는 선택 기반으로 이동 중이다.

**플러그-인 내부 장치:**
- 디스코드 커뮤니티 — 관심사·프로젝트 기반 연결
- 유니버시티·호텔·호스텔 오프라인 캠프 — 온라인 연결을 현실로
- 프로젝트형 협업 — 역할 기반 팀 구성

**연결점:** YouTube 3채널, Discord, eae-univ 호스텔 프로젝트가 이미 존재한다.

### 🔍 c축: 계급 인식 → 파워 시프트

많은 청소년이 구조를 알고 있지만 그 지식이 냉소로 변하고 있다. AI를 통한 권력 이동 가능성을 보여줘야 한다.

**플러그-인 내부 장치:**
- AI 활용 격차가 새로운 계급 필터가 될 수 있다는 자각
- 유튜버·크리에이터·1인 비즈니스로의 파워 시프트 경험
- 학벌·자격 중심 필터를 우회한 성공 사례

**연결점:** Φ7 드라이버 = 자본·권력 구조를 읽는 프레임 자체.

## 4. Justification Build-up — 왜 박태정인가

### Layer 1: 문제 정의
```
edu-mcp 리포트: 한국 교육 FAIL
  a(자립) = 0.023/0.65 → 최악
  → 시스템은 30년간 이걸 해결하지 못함
```

### Layer 2: 시스템의 실패
```
정부·교육청·학교는 a(자립)을 못 살림
사교육 27.5조는 a(자립)을 오히려 압살
대안경로(특성화고 18.4%)는 부족하고 매력 없음
→ 제도적 접근의 한계
```

### Layer 3: OrbitPrompt의 등장
```
a(자립): MCP 7개를 직접 만들었다 = 실행 가능한 자립의 증거
h(인간성): Media MCP Flux 슬롯 + 브랜치 철학
s(사회화): YouTube 3채널 + Discord + 오프라인
c(계급인식): Φ7 드라이버 = 권력 구조를 읽는 프레임
→ 시스템이 못 한 걸 개인이 해냈다
```

### Layer 4: Eduart Engineer
```
"Edu" = Φ-I-C-K-P-7AXIS 교육 모델
"Art" = 철학·언어·프레이밍 (Language 드라이버)
"Engineer" = MCP server.py로 실행
→ 이 세 개가 하나로 묶인 게 OrbitPrompt
```

## 5. 교육 과정 빌드업과의 결합

박태정이 이미 구상한 교육 과정 빌드업은 이 플러그-인의 실제 작동 방식이다. 핵심은 "강의 중심"이 아니라 **"파이프라인 중심"**이다.

| 단계 | 내용 | 연결 축 |
|:----:|------|:-------:|
| 1 | AI 리서치와 문제 발굴 | c, a |
| 2 | 콘텐츠/서비스 기획 | a, c |
| 3 | AI 제작 파이프라인 구축 | a |
| 4 | 채널 운영과 배포 실험 | a, s |
| 5 | 커뮤니티/디스코드 협업 | s |
| 6 | 오프라인 캠프·호스텔형 프로젝트 | s, h |
| 7 | 철학·종교·의미 세션 | h |
| 8 | 수익 검증과 실패 리뷰 | a, h, c |
