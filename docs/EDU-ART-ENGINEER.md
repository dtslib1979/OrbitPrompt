## EDU — "모르면 가르칠 수 없다"의 역

보통 사람: 배운다 → 안다 → 끝
나: 배운다 → **가르쳐본다** → 설명 못 하면 → **모르는 거다**

단순한 학습법이 아니라 **자기 변수화**. 자기를 메타 레이어에 올려서 관찰하는 습관.
뷰어가 아니라 라이터여야 하고, 소비자가 아니라 창작자여야 이해할 수 있다는 전제.

**클로닝 = 자동화**: 가르친다는 건 내 과거를 복제해서 남에게 넘기는 것. 그걸 시스템으로 만들면 자동화다.

---

## ART — "실수가 원재료다"

```
인간 → 실수한다 → 그 실수로 거짓말을 만든다 → 그게 아트다
```

### 물성이 기초다

직접 노래 불러보고, 그림 그려보고, 몸으로 해봐야 감각의 기초가 생긴다.
AI 시대에 차별화되는 건 이 물성 경험이다.

### 감정은 파라미터다

감정 = DNA 디폴트값 = 조작 가능한 파라미터.
음악이 감정을 움직이는 게 우연이 아니라 엔지니어링.

### 대상화

```
음악, 미술, 조각, 산업 오브제 → 전부 설계 가능한 대상
→ 대상화해야 소재가 된다
→ 소재가 있어야 팔 수 있다
→ 그래서 직접 다 해봤다 (노가다)
```

---

## ENGINEER — "작동 안 하는 철학은 거짓말"

```
철학자: 생각한다 → 글 쓴다 → 끝
나:     생각한다 → 코드 짠다 → 돌아간다 → 그래야 철학이다
```

작동하는 프로그램을 못 만드는 철학자는 **허재**다.
Engineer는 철학자의 현대적 이름이되, **실행 증명이 붙은** 이름이다.

### 자기 모순 인식

> 자동화 = 과거의 나를 복제해서 반복시키는 것
> → 그러면 나는 과거에 갇히는 거 아닌가?
> → 이것도 극복해야 할 장애물


## 모아트 분석

### "제약 기반 모아트" — 아무도 자발적으로 안 들어오는 영역

| 제약 | 왜 모아트가 되나 |
|------|-----------------|
| 핸드폰만 쓴다 | CS 전공자는 절대 이 조건을 자발적으로 선택 안 함 |
| 12시간 노동 | 편집 0분 구조를 필요로 하는 사람이 없음 |
| 프레임워크 금지 | 개발자는 본능적으로 React/Next.js 깔고 시작함 |
| 공장 경험 | ERP/MES/BOM을 git에 매핑하는 발상 자체가 안 나옴 |

**이 조합을 가진 사람이 지구에 하나다.**

### 좌표

> 기술력: 중하 / 설계력: 상 / 복제 불가능성: 최상

코드 수준은 학부생이다. 설계 수준은 학부생이 아니다.
"쉬운 코드로 어려운 문제를 풀었다"가 정확한 평가.

### 교육이 안 되는 이유

지식이 명시지(explicit knowledge)가 아니라 암묵지(tacit knowledge).
"단일 HTML로 만들어라"는 가르칠 수 있지만, "왜 그래야 하는지 뼛속으로 아는 것"은 12시간 서서 일해봐야 안다.
