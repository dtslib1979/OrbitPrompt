# 겸손·확증편향·사람 대면 — 시스템 설계 반영

> 2026-06-09
> Perplexity 분석 기반 · 박씨 경계 리스크 반영
> Media MCP 설계에 반영할 규칙

---

## 1. MCP의 목적 재정의

```
❌ "진실 발견 시스템" (위험)
✅ "주의력 낭비 줄이기 + 생각 정리할 시간 확보" (안전)
```

MCP는 왕궁이 아니라 방어벽이다.
외부 미디어의 편성권을 되찾는 도구일 뿐, "내가 만든 것이 더 똑똑하다"는 확신을 주는 시스템이 아니다.

---

## 2. 확증편향 방지 룰 (engine.py에 반영 예정)

| 룰 | 설명 | 상태 |
|-----|------|:----:|
| 반대 의견 강제 수집 | 동일 이슈 최소 1개 hostile source 없으면 risk_level=high | ⬜ |
| 좋아하는 채널 페널티 | 특정 채널 소스만 있을 때 → hold (반대 소스 들어오면 승격) | ⬜ |
| 분기별 자기검증 | 지난 분기 reject/hold 중 실제로 중요한 신호였던 사례 리뷰 | ⬜ |

---

## 3. 사람 대면 슬롯 (Flux Channel)

### 개념
MCP 파이프라인에 "사람 말 직접 듣기 전용" 슬롯을 추가.

| 항목 | 내용 |
|------|------|
| 주기 | 주 1회 이상 |
| 포맷 | 오프라인 모임 / 디스코드 / 트위터 스페이스 / 친구·가족 통화 |
| 규칙 | **그 자리에서는 편집자가 아니라 청취자로만 있음** |
| 후처리 | 나중에 생각나는 것만 사후 정리 모드로 rawmat MCP에 입력 |

### 시스템 연결
```
Flux 슬롯 (사람 대면, 청취 전용)
  → (시간 차) 필요한 것만 선별
    → rawmat MCP (사후 정리)
      → Media MCP 파이프라인
```

---

## 4. 시스템보다 사람이 위다

| 리스크 | 대응 |
|--------|------|
| "내 MCP가 더 똑똑해" | MCP 목적을 "주의력 방어벽"으로 한정 |
| 사람 목소리 = 노이즈 | Flux 슬롯 강제 (주 1회, 분석 금지, 청취만) |
| 확증편향 | 엔진 룰에 반대 의견 강제 수집 포함 |
| 오만 | 분기별 자기검증: "MCP가 틀린 케이스" 수집 |

---

## 5. 선언

> **MCP는 왕국이 아니다.**
> **멀티 LLM도, Φ7 드라이버도, 결정 엔진도 나보다 위대하지 않다.**
> **이 시스템의 목적은 진실을 찾는 것이 아니라, 내 주의력을 되찾는 것이다.**
> **사람 말을 듣는 슬롯을 막으면 시스템이 나를 고립시킨다.**
