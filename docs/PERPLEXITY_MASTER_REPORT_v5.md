## 1. 총평

OrbitPrompt는 더 이상 개인 취미 프로젝트나 프롬프트 모음집이 아니다. 이 플랫폼은 **사고를 출판하고, 판단을 실행 가능하게 만들고, 지식을 전시 가능한 형태로 재구성하는 시스템**이다.

현재 상태를 냉정하게 평가하면, 철학적 정체성은 이미 매우 강하고, 기술 구현 역시 실전 수준에 도달했다. 특히 Election MCP는 단순 데모가 아니라, 하나의 도메인 판단 체계를 MCP 서버·delta layer·gallery 전시물로 연결해 낸 첫 완성 사례다.

문제는 더 이상 "만들 수 있느냐"가 아니다. 문제는 "처음 보는 사람이 10초 안에 이 플랫폼의 가치를 이해하느냐"이다.

## 2. 현재 성취 수준

### 2-1. 이미 끝난 증명

| 항목 | 현재 상태 | 의미 |
|---|---|---|
| Election MCP v2.0 | 완료 | 단일 도메인 사고를 MCP 서버로 구조화 |
| 10개 MCP 도구 | 완료 | 계산, 연구, 분류, 시나리오, 검증 계층 확보 |
| 12개 delta layer | 완료 | 감각적 정치 해석을 제약 조건으로 박제 |
| direction_acc 100% | 완료 | 방향성 예측 능력 실증 |
| 갤러리/보드 전시 | 완료 | 결과물을 체험 가능한 형식으로 공개 |
| 사용설명서/기술명세 | 완료 | 재현과 설명의 최소 조건 확보 |

### 2-2. 핵심 강점

- 사고를 언어로 압축한다.
- 그 언어를 규칙과 제약으로 고정한다.
- 그 규칙을 MCP와 보드로 실행·전시한다.

이 구조는 앱 하나를 만드는 것보다 훨씬 강하다. 앱은 기능을 배포하지만, OrbitPrompt는 **사고방식 자체를 출판**한다.

---

## 3. OrbitPrompt의 본질 규정

> **OrbitPrompt는 인간의 사고와 도메인 판단을 MCP 기반 실행 시스템으로 번역해, 전시·교육·복제 가능한 자산으로 만드는 플랫폼이다.**

이 플랫폼은 다음과 같은 것들과 구분되어야 한다:
- 일반적인 AI 프롬프트 강의가 아니다
- 스타트업용 노코드 자동화 툴과도 다르다
- APK를 양산하는 앱 공장도 아니다
- 사람을 블랙박스로 복제하는 파인튜닝 환상도 아니다

OrbitPrompt가 하는 일은 **한 사람의 경험적 판단을 명시적 구조로 바꾸고, 그 구조를 다시 실행 가능한 MCP로 세우는 것**이다.

## 4. 타겟 규정

### 4-1. 최종 타겟 정의

> **자기 분야에서 10년 이상 일했고, 자기 경험을 시스템으로 남기고 싶으며, 최소한의 IT 리터러시를 갖춘 도메인 전문가**

### 4-2. 1차 공략군

| 우선순위 | 집단 | 이유 |
|---------|------|------|
| 1 | 제조·생산·현장 관리자 | 암묵지 비중 높음, 예외 판단 많음, 음성 설명과 잘 맞음 |
| 2 | 교육자·강사·코치 | 커리큘럼과 판단 기준을 구조화하려는 욕구 강함 |
| 3 | 법률·회계·컨설팅 | 문서와 반복 판단이 많아 MCP화에 적합 |
