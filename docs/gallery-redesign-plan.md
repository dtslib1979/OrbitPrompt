# OrbitPrompt 철학 갤러리 — 리디자인 개발 계획서

## 1. 비전: 인덱스에서 갤러리로

### 현재 문제

| 현재 (인덱스/대시보드) | 목표 (철학 갤러리) |
|------------------------|-------------------|
| 기능 중심 정보 레이아웃 | 경험 중심 전시 레이아웃 |
| 6개 섹션이 동등한 중요도 | 갤러리가 단연 중심, 나머지는 부속 |
| Gallery가 하나의 카드 그리드에 불과 | Gallery = 전체 페이지 자체 |
| Thought Module이 정보 카드 | Thought Module = 전시품 (Exhibit) |
| 다크 UI, 기능적 | 다크 UI + 무드 + 조명 + 공간감 |

### 핵심 전환

```
Before: "여기에 뭐가 있다" (카탈로그)
After:  "여기에 왜 있는지 느껴진다" (전시관)
```

## 2. 디자인 컨셉: 3개 전시관

갤러리는 3개 **Wing**(관)으로 구성. 각 Wing은 하나의 전시관이자 독립적인 경험.

### Wing A — 상설 전시관 (Permanent Exhibition)

**역할**: 출판된 Thought Module을 전시. 갤러리의 메인.

| 요소 | 설명 |
|------|------|
| 분위기 | 암전. 스포트라이트. 각 모듈이 하나의 작품처럼 |
| 전시 방식 | 카드 그리드가 아니라 **전시장 벽에 걸린 액자** 느낌 |
| 모듈 표현 | 썸네일/점수 대신 **전시 라벨**(작품명, 작가, 연도, 매체, 설명) |
| 현재 작품 | KFA Φ-I-C-K-P 시뮬레이터 = 1호 전시품 |
| 필터 | 전시관 코너(영역)별 탐색 — 축구/철학/금융/생활 |

### Wing B — 제작실 (Studio)

**역할**: Generator, FAB, PHL — 모듈을 만드는 도구들. 현재 Workshop 섹션의 진화.

| 요소 | 설명 |
|------|------|
| 분위기 | 공방/작업실. 날것의 공정 느낌 |
| 구조 | 창작 도구 카탈로그. Generator → FAB Route → PHL Protocol |
| 시각 | 배선도/회로도 스타일. 공정이 눈에 보이게 |
| 연결 | "이 Generator로 Wing A의 모듈이 만들어졌다"는 연결고리 표시 |

### Wing C — 아카이브 (Archive)

**역할**: 28개 레포 생태계 + 세션 로그 + 철학 문서. 메타 정보.

| 요소 | 설명 |
|------|------|
| 분위기 | 도서관/서고. 조용하고 집중된 |
| 구조 | 레포 생태계 맵, 세션 저널, 철학 백서, POSITION PAPER |
| 시각 | 타임라인 + 그래프 + 다이어그램 |

### 네비게이션 구조

```
┌─────────────────────────────────────────────┐
│  OrbitPrompt  │  전시관  │  제작실  │  아카이브  │
├─────────────────────────────────────────────┤
│                                             │
│         (현재 Wing의 콘텐츠)                  │
│                                             │
└─────────────────────────────────────────────┘
```

3개 Wing은 **탭/뷰 전환**으로 동작. 각 Wing이 하나의 화면을 완전히 채움.

## 3. Thought Module 전시 포맷 (Exhibit Card)

현재의 mod-card를 전시품 형식으로 업그레이드.

### Current → Target

```
현재: [아이콘] [제목] [설명] [태그] [점수] [버튼]
      → 정보 중심, 카드 UI, 기능적

목표: [전시 이미지/썸네일]
      [작품명]  [작가: 박씨]
      [연도]    [매체: Interactive Model]
      [설명]    [전시 코너: 축구]
      [관련 자료: 백서 | 모델 열기 | ParksyLog 원본]
      → 전시 라벨 느낌, 감성적, 몰입
```

## 4. 시각적 무드: 갤러리 경험

### 4.1 조명 효과

- 각 Exhibit 카드 위에 **스포트라이트** 느낌의 그라데이션 오버레이
- 마우스 위치 따라 이동하는 **미세한 헤드라이트 효과** (JS로 마우스 트래킹)
- 진입 시 **페이드 인** 애니메이션 (컬렉션 전체가 천천히 나타남)

### 4.2 타이포그래피

- 영문 전시 라벨: `Cormorant Garamond` (세리프, 갤러리 무드)
- 한국어 본문: `Inter` (가독성)
- 모노 숫자/코드: `JetBrains Mono`
- 제목: `Plus Jakarta Sans` (강조)

### 4.3 색상

| 토큰 | 역할 | 색상 |
|------|------|------|
| `--gallery-bg` | 전체 배경 | `#08080e` |
| `--gallery-wall` | 카드 배경 | `#0c0c14` |
| `--gallery-spot` | 스포트라이트 | `rgba(168,85,247,.06)` |
| `--gold` | 전시 태그 | `#d4af37` |
| `--purple` | 액센트 | `#a855f7` |
| `--cyan` | 링크/강조 | `#22d3ee` |

### 4.4 전환 효과

- Wing 전환: page transition (슬라이드 또는 fade, 400ms)
- Exhibit hover: border 글로우 + 미세 부유
- 이미지 로딩: 블러 업 (progressive loading)
- 스크롤: 패럴랙스 또는 섹션별 리빌

## 7. 핵심 디자인 원칙

1. **갤러리가 전부다** — Wing A가 메인. 진입하면 Gallery가 첫 화면.
2. **적을수록 낫다** — 한 화면에 4~6개 전시품. 무한 스크롤 금지.
3. **빛과 공간** — 어둠 속에 작품만 비춰진다. 정보는 라벨로.
4. **모든 모듈은 전시품** — 기술 블로그/README가 아니다. 전시 라벨로 써라.
5. **서사가 먼저, 기능은 다음** — "이 모듈이 왜 존재하는가"가 "무엇을 하는가"보다 먼저 보여야 한다.
6. **7축 철학과 연결** — 각 전시품은 7축 중 하나 이상과 연결. 축 간 관계가 갤러리의 맵.

## 9. 리스크와 고려사항

### 하위 호환성
- 기존 링크(`#gallery`, `#workshop`, `#fab-floor`)는 anchor 유지하거나 301
- `gallery-store.js` MODULES 배열 포맷 호환 유지
- 모든 기존 페이지 URL 변경 없음

### 성능
- 갤러리 효과는 CSS 우선, JS는 최소
- 스포트라이트/헤드라이트 효과는 `will-change: transform`으로 GPU 가속
- 폰트 3종 유지 (추가 로드 없음)

### 접근성
- 전시 라벼은 `<article>` + 적절한 heading 계층
- 애니메이션은 `prefers-reduced-motion` 대응
- 탭 전환은 키보드 내비게이션 가능

## 10. Phase 1 구현 우선순위 (바로 시작 가능)

Phase 1만으로도 현재보다 확연히 다른 갤러리 느낌.

1. **HTML 구조 변경**: 3-Wing 탭 레이아웃 (탭 네비게이션으로 섹션 분리)
2. **Exhibit 카드 포맷**: 전시품 스타일 카드로 교체
3. **CSS 토큰 추가**: 갤러리 전용 색상/효과
4. **Wing B/C**: 기존 콘텐츠를 제작실/아카이브 Wing으로 이동

Phase 2~4는 필요에 따라 순차 진행. Phase 1 완료 후 사용자 피드백으로 방향 결정.
