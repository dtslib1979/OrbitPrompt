01 — Problem + Thesis

문제는 콘텐츠 부족이 아니라 편성권 상실이다

사용자는 더 많이 보지만, 덜 통제한다. 외부 시사 방송과 해설 채널은 사건을 특정 프레임과 편성 감각으로 재포장한다. 장시간 시청은 정보 획득이라기보다 타인의 우선순위에 장시간 머무는 행위가 된다.

이 구조의 핵심 전환은 LLM 대화도 예외가 아니라는 점이다. 아이디어를 크로스체크한다는 명분으로 LLM과 길게 대화하는 행위 역시, 실시간 적응형 미디어 채널을 소비하는 것과 구조적으로 같다.

따라서 Media MCP는 뉴스 큐레이션 도구가 아니라, 외부 미디어와 LLM을 동일한 관리 대상 레이어로 재편입하는 시스템이다. 목적은 더 많이 요약하는 것이 아니라, 개인이 다시 편성권을 갖는 것이다.

시스템 명제

P1외부 미디어는 상시 복종 대상이 아니라 샘플링 대상이다.
P2LLM은 최종 권위가 아니라 역할별 제작진이다.
P3원자료는 해설보다 상위에 둔다.
P4모델 간 합의는 참고일 뿐, 진실의 근거가 아니다.
P5최종 게이트는 Python 규칙과 인간 승인자가 쥔다.
P6사용자는 필터링된 결과만 선택적으로 소비한다.
02 — Operating Model

있는 방송을 보는 구조에서 필요한 방송을 시키는 구조로

MCP는 허브, 에이전트는 제작진, 인간은 총괄 PD다.

수집
소스 에이전트가 기사, 전사문, 공식 문서, 원자료 링크를 모은다.
분석
프레임·구조 에이전트가 진영별 포장 방식과 인과 구조를 분리한다.
판정
Python 결정 레이어가 상태를 부여한다.
제작
스크립트·미디어 빌드 에이전트가 TTS → FFmpeg → 패키징을 수행한다.
보고
리포트 에이전트가 Telegram으로 결과와 상태를 전송한다.
배포
private 시청 후 인간 승인으로 public 승격.
03 — Agents + Multi-LLM Bridge

한 개의 챗봇이 아니라 하나의 편성실

Agent 01
소스
원문, 기사, 전사문, 공식 문서, 링크 수집. 원자료 우선 원칙의 출발점.
Agent 02
프레임
같은 사건을 진영별·화자별로 어떻게 포장하는지 분해. 편향 탐지 담당.
Agent 03
구조
이해관계자, 인센티브, 갈등 축, 인과관계 구조화.
Agent 04
스크립트
구조화된 재료를 TTS/내레이션용 대본으로 변환.
Agent 05
미디어 빌드
음성 합성, FFmpeg, 자막, 썸네일, 패키징, 업로드 준비.
Agent 06
리포트
상태 변화와 결과물을 Telegram 운영 인박스로 보고.
04 — Final Definition

Media MCP는 외부 미디어를 샘플 입력으로 강등하고, MCP를 오케스트레이션 허브로 세우며, LLM을 역할별 제작진으로 배치하고, Python 결정 규칙을 편성 법률처럼 삽입하고, 인간 승인 뒤에만 선택 소비와 공개 배포를 허용하는 개인 제작 운영체제다.
